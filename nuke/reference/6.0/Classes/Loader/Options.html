    The maximum number of concurrent data loading tasks. 6 by default.

  • maxConcurrentImageProcessingTaskCount

    The maximum number of concurrent image processing tasks. 2 by default.

    Parallelizing image processing might result in a performance boost in a certain scenarios, however it’s not going to be noticable in most cases. Might increase memory usage.

  • isDeduplicationEnabled

    true by default. If true loader combines the requests with the same loadKey into a single request. The request only gets cancelled when all the registered requests are.

  • isRateLimiterEnabled

    true by default. It true loader rate limits the requests to prevent Loader from trashing underlying systems (e.g. URLSession). The rate limiter only comes into play when the requests are started and cancelled at a high rate (e.g. scrolling through a collection view).

  • processor

    Returns a processor for the given image and request. By default returns request.processor. Please keep in mind that you can override the processor from the request using this option but you’re not going to override the processor used as a cache key.
